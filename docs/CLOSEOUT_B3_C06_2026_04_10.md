## 2. 行为变更

1. Save Slot 自动归档压缩：
   - 当 `fileSizeBytes >= softLimitBytes` 且 `WORLD_SAVE_SLOTS_ARCHIVE_ON_SOFT_LIMIT=true` 时，自动将 `world_save_slots.json` 归档为 `.json.gz`。
   - 默认归档目录：`tmp/world_save_slots_archive`
   - 默认保留数：`WORLD_SAVE_SLOTS_ARCHIVE_MAX_FILES=12`（超出自动清理旧归档）
2. Save Slot 多实例锁治理：
   - 持久化写入前尝试获取 `<saveSlotsPath>.lock`
   - `WORLD_SAVE_SLOTS_LOCK_STALE_MS=15000`（默认），超时锁会隔离为 `*.stale.<ts>` 后重试
   - 锁冲突时不强写，保留 `persistDirty=true` 等待后续重试
3. Save Slot 健康快照扩展：
   - 归档指标：`archiveOnSoftLimit/archiveDir/archiveMaxFiles/archiveFileCount/archiveSuccessCount/archiveFailureCount/lastArchive*`
   - 锁指标：`lockPath/lockStaleMs/lockContentionCount/lockStealCount/lockFailureCount`
4. runtime 与 nightly 告警扩展：
   - `save_slots_archive_failures`
   - `save_slots_lock_contention`
   - `save_slots_lock_failures`
5. nightly 新增检查：
   - `save_slots_archive_failures_absent`
   - `save_slots_lock_contention_absent`
   - `save_slots_lock_failures_absent`
