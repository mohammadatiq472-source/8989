# 原生 SLG 页面结构（2026-04-16）

> 状态：附录快照。  
> 页面结构现行口径请改读：[NATIVE_SLG_MAINLINE_INDEX.md](NATIVE_SLG_MAINLINE_INDEX.md) 和 [NATIVE_SLG_COMPONENT_ARCHITECTURE.md](NATIVE_SLG_COMPONENT_ARCHITECTURE.md)

> 目标：把客户端页面结构从 `preview/story` 口径收回到原生 SLG 口径，形成后续 Godot 主入口和主界面重建的硬骨架。

## 1. 页面结构总则

客户端页面不按 demo 页面拆，而按“持续壳层 + 主模式 + 功能覆盖层”组织。

核心原则：

1. 玩家默认停留在主城，不停留在预览页。
2. 大地图是同一主壳层下的模式切换，不是孤立页面。
3. 武将、招募、同盟、势力、战报等都应是功能覆盖层，不应再复制一套世界结构。
4. AI 入口必须嵌在原生 SLG 主流程里，而不是独立演示系统。

## 2. 四层结构

### 2.1 持续壳层

所有主模式共享同一套常驻 UI：

- 顶部全局状态条
- 左侧任务/战报/邮件抽屉流
- 底部主功能栏
- 右上世界层级切换入口

持续壳层负责：

- 保持玩家对资源、计时、状态的连续感知
- 保持主功能入口位置稳定
- 承载主城与大地图之间的切换

### 2.2 主城模式

主城是默认落点，也是第一阶段最该先做好的页面。

主城模式最低组成：

- 中央主城舞台
- 建筑或功能节点的可交互区
- 主城内的产出与反馈
- 主城内的上下文动作面板

主城模式的设计要求：

1. 玩家进入游戏后先看到主城，而不是故事沙箱。
2. 主城是操作面，不是纯视觉背景。
3. 从主城可以直接进入武将、内政、招募、同盟、AI中枢。
4. 从主城可以稳定进入大地图，并能返回主城原状态。

### 2.3 大地图模式

大地图是世界态的主操作页，但仍属于同一壳层。

大地图最低组成：

- 左侧模式栏：国家 / 州 / 战区 / 地块层级
- 左中上下文信息面板：选中对象后的详情与动作
- 中央主地图：LOD 递进
- 右侧工具条：缩放、筛选、跳转、定位
- 底部世界动作条：坐标、行军、驻守、视野、切换

大地图的设计要求：

1. 支持从宏观到微观的空间层级递进。
2. 州、城、地块切换要共享同一视觉语法。
3. 大地图返回主城时，不丢失壳层与路径关系。
4. 大地图中的动作区只在有上下文时出现，不永久占屏。

### 2.4 功能覆盖层

以下页面统一视为覆盖层，而不是新的根页面：

- 武将
- 招募
- 同盟
- 势力
- 战报
- 内政

覆盖层的要求：

1. 从主城或大地图进入。
2. 共享顶部结构与关闭逻辑。
3. 返回时回到进入前的上下文。
4. 信息密度高于主场景，但不改变世界模式关系。

## 3. 核心槽位定义

### 3.1 顶部全局状态条

固定内容：

- 头像
- 资源
- 货币
- 网络
- 邮件或倒计时

要求：

- 主城与大地图都保持一致
- 不因为切页而改布局
- 后续可逐步替换美术，但结构先稳定

### 3.2 左侧功能流

承载：

- 任务
- 战报
- 邮件
- 事件提示

要求：

- 以抽屉或窄流方式存在
- 不压住中央主舞台
- 只在需要时展开，不常驻厚面板

### 3.3 底部主功能栏

第一阶段固定为：

`武将 / 内政 / 同盟 / 国战 / AI中枢 / 招募 / 背包 / 设置`

要求：

1. 位置稳定，不随模式变动。
2. `AI中枢` 固定占底栏中部强入口。
3. 即使暂时没有完整美术，也先用结构占位与简称完成。

### 3.4 右上世界入口

固定保留 `大地图` 切换入口。

要求：

- 位置稳定
- 从主城可直接切世界态
- 从世界态可直接回主城

## 4. 页面优先级

当前实现顺序应固定如下：

1. 持续壳层
2. 主城模式
3. 大地图模式
4. 功能覆盖层

不建议的顺序：

- 先继续做 preview story
- 先继续做 map_surface 小语义收口
- 先做过多覆盖层页面
- 先做复杂 AI 专属 UI

## 5. 与当前仓库的对应关系

这份结构文档对应的是“目标界面骨架”，不是现状描述。

现状里可直接复用或借鉴的主要是：

- `server/**` 的世界与会话主干
- `shared/**` 的共享契约
- `godot-client/scripts/map/**` 的地图渲染与空间表达底座

现状里不应继续当作主页面结构推进的是：

- `godot-client/data/ui_preview/stories/**`
- `godot-client/scripts/dev/**`
- `godot-client/scenes/dev/**`

## 6. 对后续代码改造的硬约束

如果后续代码改造不能服务下面任一项，就说明仍然偏航：

1. 是否在重建持续壳层？
2. 是否在重建主城作为默认驻留页？
3. 是否在把大地图收回为同壳层下的模式切换？
4. 是否在把功能页压回覆盖层，而不是继续加 demo 场景？
