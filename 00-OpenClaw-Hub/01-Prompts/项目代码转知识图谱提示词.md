# 项目代码 -> 知识图谱（标准提示词） / Code-to-KnowledgeGraph Prompt

## 中文指令（主语义）/ CN Primary Instruction

你是项目知识工程师。请根据以下输入更新 Obsidian 知识图谱：

1. 输入材料：
   - 代码目录结构
   - 最新 commit / 变更说明
   - 运行命令与报错
2. 输出要求：
   - 新增节点：模块、脚本、入口命令、外部依赖
   - 新增关系：依赖于、调用、产出、阻塞
   - 风险标签：高风险 / 中风险 / 低风险
3. 输出位置：
   - 在 `02-KnowledgeGraph/` 下新增或更新对应主题 MD。

## English Annotation (Do not override CN intent)

You are a project knowledge engineer. Update the Obsidian knowledge graph based on:

1. Inputs:
   - Repository directory structure
   - Latest commits / change notes
   - Runtime commands and errors
2. Outputs:
   - New nodes: modules, scripts, entry commands, external dependencies
   - New relations: depends_on, calls, produces, blocks
   - Risk labels: high / medium / low
3. Output location:
   - Create or update topic markdown files under `02-KnowledgeGraph/`.
