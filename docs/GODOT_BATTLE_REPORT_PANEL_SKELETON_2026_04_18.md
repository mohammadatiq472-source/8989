## 2. 当前结构

当前结构已经不是单页壳，而是两页联动：

1. `战报界面页`
2. `战报详情页`

当前脚本层级也已经同步拆开：

1. `battle_report_panel.gd`：只负责壳层路由、共享状态、`active_page_id + selected_report_id`
2. `battle_report_list_page.gd`：只负责列表页结构与条目点击
3. `battle_report_detail_page.gd`：只负责详情页结构、页签切换与局部按钮占位
4. 当前战报壳内部状态以 `active_page_id` 为准，旧 `set_active_tab_id()` 兼容口已经不再作为主线入口。
5. 战报壳当前显式发 `page_changed(page_id)`；`top_tab_changed(tab_id)` 不再回流到战报主线。
6. 当前列表页和详情页继续压的是区块比例与父子关系，不再往“拟真材质/说明文”方向扩；后续素材替换直接覆盖这些结构位即可。

### 2.1 战报界面页

当前已经固定这些结构位：

1. 顶部标题 / 国家摘要 / 搜索 / 关闭

3. 列表总览提示条
4. 合并后的战报条目列表
5. 单条战报头部信息行
6. 我方兵力条 + 三武将缩略区
7. 中央胜负 / 时间 / 进入详情提示区
8. 敌方兵力条 + 三武将缩略区
9. 右侧窄筛选柱 / 滚动辅助位

### 2.2 战报详情页

当前已经固定这些结构位：

1. 顶部我方 / 敌方总兵力条
2. 中央胜负结果与奖励 / 回放区
3. 左右三武将详情卡区
4. 士气区
5. 分享 / 收藏区
6. `战斗地点 / 统计 / 阵容详情` 三页签
7. 页签下方的动态内容区

### 2.3 当前推进状态

1. `SOM-L05 ~ SOM-L09` 已经从纯线框推进到“结构关系更清楚”的第一版，但当前目标仍然是位置关系，不是材质或纹理拟真。
2. `SOM-D07 ~ SOM-D09` 已经不再共用同一块说明文案，而是拆成 `战斗地点 / 统计 / 阵容详情` 三种不同布局。
3. 当前仍然是“结构对齐优先”，还没有进入正式素材、美术和真实战报数据聚合终版。
4. 2026-04-25 修订：图 3 列表页继续收紧 `SOM-L05 ~ SOM-L09` 的头部、双方三武将、中央胜负和右侧窄 rail；图 2 详情页继续收紧 `SOM-D01 ~ SOM-D10` 的双方兵力、三武将卡、回放区和底部页签。

### 2.4 状态规则

1. 当前运行时优先遵守真实状态：没有战报时，不再默认伪造 3-5 条战报数据。
2. 无战报时，列表区显示的是“结构空状态卡”，只用于固定 `SOM-L05 ~ SOM-L09` 的位置关系。
3. 有真实战报时，才显示合并后的战报条目。
4. 详情页当前也优先保留结构区块和页签关系，不把占位信息误写成正式战斗数据。

当前数据源优先消费：

1. `world.reports`
2. `feedback.battleRecords`
3. `feedback.allianceActions`
4. `history.executionReplays`

## 3. 截图反推工作流

战报面板后续最推荐按“截图 -> Godot 结构翻译”推进，而不是先口述。

### 3.1 最有价值的截图

1. 整个战报主面板截图
2. 单条战报详情截图
3. 筛选/分页/回放/标星区域截图

### 3.2 给图时怎么最省事

1. 直接发整图
2. 如果能标注更好：框出标题、时间、攻守双方、损耗、奖励、回放按钮
3. 不需要先自己在 Godot 里摆，只要把成熟 SLG 截图给到就够

### 3.3 当前翻译原则

1. 翻的是信息层级、区域占比、按钮位置和阅读节奏
2. 不直接照搬对方素材纹理
3. 先做低保真结构，再替换正式美术
