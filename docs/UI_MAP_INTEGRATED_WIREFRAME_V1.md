## 3. 地图层级与缩放蓝图（10万格）

### 3.1 地图规模基线

1. 地图总格子：约 `100,000`（建议起始实现：`320 x 320 = 102,400`）。
2. 微观地块：用于逐块推进、攻打、扫荡、占领、资源采收。
3. 地图拖拽：默认启用鼠标/触控拖拽 + 滚轮/双指缩放。

### 3.2 LOD（缩放分层）

1. `L0-国家层`（最远）：仅显示国家颜色块、国号、国力、首都城池。
2. `L1-州层`：显示东汉十三州边界、州中心城池、州际关口。
3. `L2-战区层`：显示州内多个战区（以城池为中心分区）、热点和压强。
4. `L3-地块层`（最近）：显示单地块细节，支持攻打/扫荡/占领/采收。

### 3.3 交互规则

1. 缩小时自动合并视觉为州/战区信息，不渲染全部细粒度标签。
2. 放大到地块层后，显示可行动作与路径细节。
3. 所有作战操作最终都在 `L3-地块层` 下单执行。

## 5. 山脉、河流、关卡规则（州际阻隔）

### 5.1 地形骨架

1. 山脉：太行、秦岭、巴山、南岭、武夷等作为天然阻隔带。
2. 水系：黄河、长江、淮河、汉水、珠江水系作为跨州边界参考。
3. 阻隔效果：提升通行消耗，限制大部队穿越路径，迫使争夺关口。

### 5.2 关卡机制

1. 每个州与邻州之间配置 2~4 个主关口（可按历史地理命名）。
2. 未占领关口前，不可大规模推进至下一州核心区域。
3. 关口可升级防御，成为国战焦点节点。

## 6. 战区划分规则（以城池为中心）

1. 战区由“中心城池 + 周边地块簇”组成。
2. 每州至少 1 个州会城，多个县级城池作为战区锚点。
3. 战区内配置资源地、关口、据点，形成“发育 -> 前推 -> 固守”循环。
4. 战区是后续立国和同盟控制判定的最小战略单元。

## 7. 国家系统蓝图（立国、颜色、国号）

### 7.1 立国条件（建议）

1. 同盟完整控制至少 1 个战区。
2. 必须控制该战区中心城池。
3. 满足最低资源与活跃成员条件（防止早期刷立国）。

### 7.2 立国流程

1. 在右上角 `大地图` 中进入“立国”面板。
2. 选择国号。
3. 选择国家颜色。
4. 确认首都城池（默认当前占领的战区中心城池）。

### 7.3 大地图展示

1. 国家颜色填充其控制区域。
2. 点击国家可查看：同盟名、国号、实力值、首都城池、战区数量。
3. 支持按州/战区筛选查看势力分布。

### 8.2 多 AI 将军（发育开荒）

1. 支持为多个 AI 将军分配不同战区与职责。
2. 职责建议：主攻、守备、侦察、后勤、开荒。
3. 可设置并发执行上限，避免同一资源地过度冲突。
4. 可设置“立国优先模式”，AI 自动优先抢占目标战区核心城。

### 8.3 模型动态选择

1. 模型列表由后端动态拉取中转站可用模型。
2. 可分层配置 Commander / General / Unit 模型。
3. 拉取失败时回退上次缓存并允许手动输入模型 ID。

### 8.4 提示词配置

1. 全局 Doctrine（全局战略）。
2. 将军角色提示词（中层策略）。
3. 单次战役补充指令（短期目标）。
4. 预设模板：稳守、扩张、侦察优先、立国优先。

## 9. 右上角“大地图”按钮详设

1. 入口位置：主界面右上角，常驻。
2. 面板功能：
   - 层级切换：国家层 / 州层 / 战区层 / 地块层。
   - 势力筛选：按同盟、国家、州过滤。
   - 信息悬浮：显示国家颜色、国号、实力、首都城池。

## 10. 页面线框清单（浮层，不离开地图）

### 10.1 左侧任务流（任务/活动/战报/邮件）

1. 入口位置：左侧竖列按钮。
2. 交互方式：点击后左侧展开抽屉浮层。
3. 页面结构：分类切换 + 列表 + 快捷操作。
4. 状态要求：空状态、加载状态、失败重试状态。

### 10.2 右侧地块动作区（核心战斗闭环）

1. 触发方式：点击任意地块。
2. 页面结构：地块卡 + 情报条 + 动作按钮 + 二次确认。
3. 执行后反馈：箭头动画 + 队列状态 + 战报写入。
4. 资源地占领后：自动进入资源产出并在内政可见。

### 10.3 底栏功能浮层

1. 武将：编组、替换、回城、驻防。
2. 内政：升级、加速、收取、分配资源。
3. 同盟：发布指令、战区协防、成员协同。
4. 国战：关卡推进、战区争夺、州际战线。
5. AI中枢：多 AI 托管、模型、提示词、日志。
6. 招募：卡池、保底、预览。
7. 背包：道具使用与批量操作。
8. 设置：画质、操作、性能模式。

## 17. 战斗公式武将属性集成（2026-03-19）

`resolveBattleAtTile`（`shared/domain/rules.ts`）现已引入武将三维属性：

| 属性 | 变量名 | 范围 | 基准值 | 效果 |
|------|--------|------|--------|------|
| 统率 | `hero.command` | 40–98 | 60 | 提升兵力发挥效率（±0.4%/点） |
| 机动 | `hero.agility` | 40–98 | 60 | 提升机动力贡献（±0.6%/点） |
| 谋略 | `hero.tactics` | 40–98 | 60 | 降低己方伤亡（±0.5%/点） |

公式修改点：
- `attackerPower` 中 strength 项乘以 commandBonus，mobility 项乘以 agilityBonus
- `defenderPower` 同理
- 损失计算乘以 tacticsReduction（攻防双方各自生效）
