# 收尾总览（B2-C21，2026-04-10）

## 0. 结论

`B2-C21` 已完成：模板回放链路升级为“固定初始态夹具 + 全必选步骤 + 默认可跑通”。

---

## 1. 代码改动

1. 固定初始态回放夹具（后端）：
   - `server/src/application/world/WorldService.ts`
   - 新增 `primeReplayFixtureSlot(slotId, { label, source })`，支持 `initial_world_v1/current_world` 固定装载源。
2. 夹具 API（后端路由）：
   - `server/src/routes/observability.ts`
   - `server/src/app.ts`
   - 新增 `POST /api/save-slots/fixture/prime`。
3. 主线 gate 收紧：
   - `server/src/evals/runAiMainlineStabilityGate.ts`
   - 回放前强制 `backup -> prime(initial_world_v1) -> load fixture`，回放后强制 restore backup。
4. CLI 回放收紧：
   - `godot-client/tools/slg_ops_cli.py`
   - `template-replay` 新增夹具参数：`--fixture-slot-id`、`--backup-slot-id`、`--fixture-source`、`--restore-world`
   - 模板步骤全部 `required=true`（不再保留可选步骤）
   - 回放内链超时下限提升为 `60s`，避免 `advance_tick` 长耗时误判。

---

## 2. 关键行为变化

1. 回放默认不再依赖“当前世界随机态”，改为固定初始态种子：
   - 默认 `--fixture-source initial_world_v1`
2. 回放步骤收口为全必选：
   - `clear_plan_execution`
   - `move_first_unit`
   - `tactical_override_first_unit`
   - `preview_national_agenda`
   - `preview_court_session`
   - `advance_tick`
3. 失败保护：
   - 夹具引导失败时会尝试 restore backup，并输出 `fixture_bootstrap_failed` 报告。

## 4. 风险与后续

1. 风险：`advance_tick` 在某些机器/配置下耗时可接近 50s，继续保留 60s 超时下限是必要保护。
2. 后续建议：把 `template-replay` 固化进 nightly 汇总卡，形成“session-security + mainline + fixture-replay”三件套连续回归。
